## சுருக்கம்

இந்த ஆய்வுக்கட்டுரை **Legalis-RS**-ஐ முன்வைக்கிறது, இது இயற்கை மொழி சட்ட ஆவணங்களை **நிர்ணயிக்கக்கூடிய தர்க்கம் (Code)** மற்றும் **நீதித்துறை விருப்புரிமை (Narrative)** என கண்டிப்பாக பிரிக்கும் மற்றும் கட்டமைக்கும் Rust கட்டமைப்பு.

நவீன சட்ட அமைப்புகள் கணினி தானியங்கிமயமாக்கல் செய்யக்கூடிய துறைகள் (வயது தேவைகள், வருமான வரம்புகள், காலக்கெடு கணக்கீடுகள்) மற்றும் மனித விளக்கம் மற்றும் தீர்ப்பு தேவைப்படும் துறைகள் ("நியாயமான காரணம்", "பொது ஒழுக்கம்") ஆகியவற்றின் கலவையைக் கொண்டுள்ளன. முந்தைய அணுகுமுறைகள் இந்த எல்லையை தெளிவற்றதாக விட்டுவிட்டன அல்லது எல்லாவற்றையும் கணக்கிடக்கூடியதாக மாற்ற முயற்சித்த அதிகப்படியான தானியங்கிமயமாக்கலை முயற்சித்தன.

Legalis-RS மூன்று-மதிப்பு தர்க்க வகை `LegalResult<T>`-ஐ அறிமுகப்படுத்துகிறது, இது வகை மட்டத்தில் இந்த எல்லையை வெளிப்படையாக மாற்ற Rust-ன் வகை அமைப்பைப் பயன்படுத்துகிறது. இது AI சகாப்தத்தில் "அல்காரிதம் சர்வாதிகாரத்தை" தடுக்கும் அதே வேளையில் சட்ட பிழைத்திருத்தம், உருவகப்படுத்துதல் மற்றும் சர்வதேச இடமாற்றத்திற்கான புதிய மாதிரியை செயல்படுத்துகிறது.

**முக்கிய தொழில்நுட்ப பங்களிப்புகள்**:
1. சட்ட டொமைன் குறிப்பிட்ட மொழி (DSL) மற்றும் பாகுபடுத்தி செயலாக்கம்
2. Z3 SMT solver மூலம் முறையான சரிபார்ப்பு
3. சமூக தாக்கத்தை கணிக்க ECS-பாணி உருவகப்படுத்துதல் இயந்திரம்
4. 25+ blockchain தளங்களுக்கு ஸ்மார்ட் ஒப்பந்த உருவாக்கம்
5. செமாண்டிக் வலைக்கான Linked Open Data (RDF/TTL) ஒருங்கிணைப்பு
6. கலாச்சார அளவுரு தழுவலுடன் (Soft ODA) 4 நாடுகளுக்கான சட்ட அமைப்பு செயலாக்கங்கள்

**முக்கிய தத்துவம்**: *"எல்லாமே கணக்கிடக்கூடியதாக இருக்க வேண்டியதில்லை."*

## 1. அறிமுகம்

### 1.1 பின்னணி: சட்டம் மற்றும் கணக்கீட்டுக்கு இடையிலான உறவு

Lawrence Lessig-ன் புகழ்பெற்ற ஆய்வுக்கட்டுரை "Code is Law" சைபர்வெளியில் கட்டமைப்பு (குறியீடு) சட்டத்திற்கு சமமான ஒழுங்குமுறை அதிகாரத்தைக் கொண்டுள்ளது என்று சுட்டிக்காட்டியது. இருப்பினும், Legalis-RS இதை மாற்றுகிறது, "**சட்டம் குறியீடாக மாறுகிறது**" என்ற அணுகுமுறையை ஏற்றுக்கொள்கிறது.

சட்டத்தை குறியீடாக மாற்றுவது பின்வரும் நன்மைகளை வழங்குகிறது:

- **சரிபார்க்கக்கூடிய தன்மை**: தொகுப்பு நேரத்தில் தர்க்க முரண்பாடுகளைக் கண்டறிதல்
- **உருவகப்படுத்துதல்**: அமலாக்கத்திற்கு முன் சமூக தாக்கங்களை கணித்தல்
- **இயங்குதன்மை**: வெவ்வேறு சட்ட அமைப்புகளுக்கு இடையில் மாற்றம் மற்றும் ஒப்பீடு
- **வெளிப்படைத்தன்மை**: சட்ட முடிவு செயல்முறைகளின் முழுமையான தணிக்கை தடங்கள்

இருப்பினும், எல்லா சட்டங்களையும் கணக்கிடக்கூடியதாக மாற்றுவது தத்துவ ரீதியாகவும் நடைமுறையிலும் ஆபத்தானது. சட்டம் இயல்பாகவே "மனித தீர்ப்பு" தேவைப்படும் துறைகளைக் கொண்டுள்ளது, இதை புறக்கணிக்கும் தானியங்கிமயமாக்கல் "AI சர்வாதிகாரத்திற்கு" வழிவகுக்கலாம்.

### 1.2 பிரச்சனை அறிக்கை: AI சகாப்தத்தில் சட்ட செயலாக்கத்தின் சவால்கள்

நவீன சட்ட தொழில்நுட்பம் (LegalTech) பல அடிப்படை சவால்களை எதிர்கொள்கிறது:

1. **தெளிவின்மை கையாளுதல்**: பல சட்ட சொற்கள் வேண்டுமென்றே தெளிவற்றவை, வழக்கு-வழக்காக விளக்கத்தை கருதுகின்றன
2. **சூழல் சார்பு**: அதே விதி சமூக மற்றும் கலாச்சார சூழலைப் பொறுத்து வேறுபட்ட முறையில் விளக்கப்படலாம்
3. **காலப்போக்கில் மாற்றம்**: சட்டங்கள் திருத்தப்பட்டு ரத்து செய்யப்படுகின்றன, காலம் முழுவதும் நிலைத்தன்மை மேலாண்மை தேவைப்படுகிறது
4. **சர்வதேச வேறுபாடுகள்**: ஒவ்வொரு நாட்டின் சட்ட அமைப்புகளும் தங்கள் தத்துவ அடித்தளத்திலிருந்து வேறுபடுகின்றன

### 1.3 முன்மொழிவு: கணக்கிடக்கூடிய தன்மை மற்றும் நீதித்துறை விருப்புரிமையின் பிரிப்பு

Legalis-RS-ன் மையம் `LegalResult<T>` வகை மூலம் மூன்று-மதிப்பு தர்க்கத்தின் அறிமுகமாகும்:

```rust
pub enum LegalResult<T> {
    /// [நிர்ணயிக்கக்கூடிய துறை] தானியங்கி செயலாக்கம் சாத்தியமான சட்ட முடிவுகள்
    Deterministic(T),

    /// [விருப்புரிமை துறை] மனித தீர்ப்பு தேவைப்படும் துறை
    JudicialDiscretion {
        issue: String,           // கையில் உள்ள பிரச்சனை
        context_id: Uuid,        // சூழல் தரவு
        narrative_hint: Option<String>, // LLM மூலம் குறிப்பு கருத்து
    },

    /// [தர்க்க முறிவு] சட்டத்திலேயே உள்ள பிழை
    Void { reason: String },
}
```

இந்த வகை சட்ட செயலாக்கத்தின் முடிவு எப்போதும் மூன்று வகைகளில் ஒன்றாக வகைப்படுத்தப்படுவதை உறுதி செய்கிறது. `JudicialDiscretion` அடையும்போது அமைப்பு செயலாக்கத்தை நிறுத்தி மனிதர்களுக்கு தீர்ப்பை ஒப்படைக்கிறது. இது AI சர்வாதிகாரத்திற்கு எதிரான "வகை-நிலை கோட்டையாக" மாறுகிறது.

## 2. தொடர்புடைய பணிகள்

### 2.1 கணக்கீட்டு சட்டத்தின் வரலாறு

சட்டம் மற்றும் கணினிகளுக்கு இடையிலான உறவு 1950களில் LARC திட்டத்திலிருந்து தொடங்குகிறது.

| சகாப்தம் | தொழில்நுட்பம் | பண்புகள் |
|---------|--------------|---------|
| 1950கள் | LARC | முதல் சட்ட தகவல் மீட்டெடுப்பு அமைப்பு |
| 1970கள் | MYCIN-வகை நிபுணர் அமைப்புகள் | விதி-அடிப்படையிலான பகுத்தறிவு |
| 1980கள் | HYPO | வழக்கு-அடிப்படையிலான பகுத்தறிவு |
| 1990கள் | XML/SGML தரப்படுத்தல் | சட்ட ஆவணங்களை கட்டமைத்தல் |
| 2000கள் | செமாண்டிக் வலை | ஒன்டாலஜி-அடிப்படையிலான சட்ட அறிவு பிரதிநிதித்துவம் |
| 2010கள் | இயந்திர கற்றல் | சட்ட கணிப்பு மாதிரிகள் |
| 2020கள் | LLM + முறையான சரிபார்ப்பு | கலப்பின அணுகுமுறை |

### 2.2 இந்த திட்டத்தின் நிலைப்படுத்தல்

Legalis-RS தற்போதைய ஆராய்ச்சியை பின்வரும் வழிகளில் விரிவுபடுத்துகிறது:

1. **வகை-நிலை விருப்புரிமை குறிப்பிடுதல்**: `LegalResult<T>` மூலம் மூன்று-மதிப்பு தர்க்கம்
2. **ஒருங்கிணைந்த கட்டமைப்பு**: Parse→Verify→Simulate→Output குழாய்
3. **பல-வடிவ இயங்குதன்மை**: Catala/L4/Stipula/Akoma Ntoso உடன் மாற்றம்
4. **சர்வதேசமயமாக்கல் வடிவமைப்பு**: கலாச்சார அளவுரு தழுவல் (Soft ODA)
5. **Blockchain ஒருங்கிணைப்பு**: 25+ தளங்களுக்கு ஸ்மார்ட் ஒப்பந்த உருவாக்கம்

## 3. தத்துவம் & வடிவமைப்பு கொள்கைகள்

### 3.1 "குறியீட்டாக ஆளுகை, கதையாடலாக நீதி"

Legalis-RS-ன் கோஷம் ஆளுகை மற்றும் நீதிக்கு இடையிலான அடிப்படை வேறுபாட்டை பிரதிபலிக்கிறது:

- **ஆளுகை**: விதி பயன்பாடு, நடைமுறை இணக்கம், தகுதி தீர்மானம் → **குறியீடாக மாற்றக்கூடியது**
- **நீதி**: சமத்துவம் உணர்தல், சூழல் விளக்கம், மதிப்பு தீர்ப்பு → **கதையாடலாக சொல்லப்படுவது**

### 3.2 மூன்று-மதிப்பு தர்க்க வடிவமைப்பு

`LegalResult<T>`-ன் மூன்று மதிப்புகள் பின்வரும் சட்ட தத்துவ கருத்துக்களுடன் ஒத்துப்போகின்றன:

| வகை | சட்ட தத்துவ கருத்து | செயலாக்க முகவர் |
|------|---------------------|----------------|
| `Deterministic(T)` | இயந்திரமாக பயன்படுத்தக்கூடிய விதிகள் | கணினி |
| `JudicialDiscretion` | விளக்கம் தேவைப்படும் கொள்கைகள் | மனிதன் |
| `Void` | சட்ட இடைவெளிகள்/முரண்பாடுகள் | சட்டமன்றம் (திருத்தம் தேவை) |

### 3.3 "எல்லாமே கணக்கிடக்கூடியதாக இருக்க வேண்டியதில்லை"

எல்லாவற்றையும் கணக்கிடக்கூடியதாக மாற்றும் சோதனைக்கு எதிராக, Legalis-RS தெளிவாக "இல்லை" என்கிறது. பின்வரும் துறைகள் வேண்டுமென்றே கணக்கிட முடியாததாக வடிவமைக்கப்பட்டுள்ளன:

1. **நியாயமான காரணம்**
2. **பொது ஒழுங்கு மற்றும் ஒழுக்கம்**
3. **நல்லெண்ணம்**
4. **நியாயத்தன்மை**

### 3.4 AI சர்வாதிகாரத்தை தடுத்தல்

Legalis-RS பின்வரும் வழிமுறைகள் மூலம் AI சர்வாதிகாரத்தை தடுக்கிறது:

1. **வகையால் கட்டாய நிறுத்தம்**: `JudicialDiscretion` அடையும்போது தானியங்கி நிறுத்தம்
2. **கட்டாய தணிக்கை தடங்கள்**: அனைத்து முடிவு செயல்முறைகளின் பதிவு
3. **விளக்கக்கூடிய தன்மை**: முடிவு காரணங்களின் கட்டமைக்கப்பட்ட வெளியீடு
4. **மனித வளையம் உத்தரவாதம்**: விருப்புரிமை துறைகளில் மனிதர்கள் எப்போதும் இறுதி முடிவுகளை எடுக்கிறார்கள்

### 4.2 மைய அடுக்கு

#### legalis-core
திட்டத்தின் தத்துவ மையத்தை செயல்படுத்தும் crate.

**முக்கிய வகை வரையறைகள்**:
- `LegalResult<T>`: மூன்று-மதிப்பு தர்க்க வகை
- `Statute`: சட்டங்களின் அடிப்படை பிரதிநிதித்துவம்
- `Condition`: நிபந்தனை வெளிப்பாடுகள் (AND/OR/NOT, வயது, வருமானம், முதலியன)
- `Effect`: சட்ட விளைவுகள் (Grant/Revoke/Obligation/Prohibition)

#### legalis-dsl
சட்ட டொமைன் குறிப்பிட்ட மொழிக்கான பாகுபடுத்தி.

**DSL தொடரியல் எடுத்துக்காட்டு**:
```
STATUTE adult-voting: "வயது வந்தோர் வாக்குரிமை" {
    JURISDICTION "IN"
    VERSION 2
    EFFECTIVE_DATE 2024-01-01

    WHEN AGE >= 18 AND HAS citizen
    THEN GRANT "வாக்குரிமை"

    DISCRETION "மனநல திறன் தீர்மானம் மருத்துவ நோய் கண்டறிதல் தேவை"
}
```

## 5. மைய தொழில்நுட்பங்கள்

### 5.1 சட்ட DSL

**அடிப்படை கட்டமைப்பு**:
```
STATUTE <id>: "<title>" {
    [JURISDICTION "<jurisdiction>"]
    [VERSION <number>]
    [EFFECTIVE_DATE <date>]

    WHEN <condition>
    THEN <effect>

    [EXCEPTION WHEN <condition>]
    [DISCRETION "<description>"]
}
```

### 5.2 LegalResult<T> வகை

```rust
pub enum LegalResult<T> {
    Deterministic(T),
    JudicialDiscretion {
        issue: String,
        context_id: Uuid,
        narrative_hint: Option<String>,
    },
    Void { reason: String },
}
```

### 5.3 Z3 SMT Solver மூலம் முறையான சரிபார்ப்பு

**சரிபார்ப்பு இலக்குகள்**:
1. வட்ட குறிப்புகள்
2. அணுக முடியாத சட்டங்கள்
3. தர்க்க முரண்பாடுகள்
4. அரசியலமைப்பு மோதல்கள்

### 5.4 ECS-பாணி உருவகப்படுத்துதல் இயந்திரம்

உருவகப்படுத்துதல் இயந்திரம் Entity-Component-System (ECS) வடிவத்தை ஏற்றுக்கொள்கிறது:
- **Entity**: குடிமக்கள் முகவர்கள்
- **Component**: பண்புகள் (வயது, வருமானம், வசிப்பிடம், முதலியன)
- **System**: சட்ட பயன்பாட்டு தர்க்கம்

## 7. வழக்கு ஆய்வுகள்

### 7.1 நலன்புரி நன்மை தகுதி தீர்மான அமைப்பு

**முடிவுகள்**:
- **நிர்ணயிக்கக்கூடிய முடிவுகள்**: 85% வழக்குகள்
- **JudicialDiscretion**: 15% வழக்குகள்

### 7.2 சிவில் சட்டம் பிரிவு 709 (தவறான செயல்) உருவகப்படுத்துதல்

**சோதனை காட்சிகள்**:
1. தெளிவான வேண்டுமென்றே தவறான செயல் → `Deterministic(Liable)`
2. கவனக்குறைவால் தவறான செயல் → `Deterministic(Liable)`
3. எல்லை வழக்கு → `JudicialDiscretion`

## 11. முடிவு

Legalis-RS வகை அமைப்பில் "கணக்கிடக்கூடிய தன்மை மற்றும் மனித தீர்ப்புக்கு இடையிலான எல்லையை" வெளிப்படையாக மாற்றுவதன் மூலம் சட்டத்தை குறியீடாக மாற்றுவதற்கான புதிய அணுகுமுறையை முன்வைக்கிறது.

**முக்கிய சாதனைகள்**:

1. **தத்துவ அடித்தளம்**: "குறியீட்டாக ஆளுகை, கதையாடலாக நீதி"
2. **வகை அமைப்பு**: `LegalResult<T>` மூலம் மூன்று-மதிப்பு தர்க்கம்
3. **ஒருங்கிணைந்த கட்டமைப்பு**: 7 அடுக்குகள் மற்றும் 16 crates கொண்ட விரிவான வடிவமைப்பு
4. **செயலாக்கம்**: சுமார் 450,000 வரிகள் Rust குறியீடு
5. **சரிபார்ப்பு**: Z3 SMT solver ஒருங்கிணைப்பு
6. **உருவகப்படுத்துதல்**: ECS-பாணி இயந்திரம் (GPU முடுக்க ஆதரவு)
7. **வெளியீடு**: 25+ blockchains, RDF/TTL, பல வடிவங்கள்

**முக்கிய தத்துவம்**: *"எல்லாமே கணக்கிடக்கூடியதாக இருக்க வேண்டியதில்லை."*
